运行环境
Xia Lv3
 2021-04-29 01:40:13  
  • 面试题 
 
  • 运行环境即浏览器（server 端有 node.js）
  • 下载网页代码，渲染出页面，期间会执行若干 JS
  • 要保证代码在浏览器中：稳定且高效

1. 网页加载过程

1.1 资源的形式

  • HTML 代码
  • 媒体文件，如图片、视频等
  • JavaScript、CSS

1.2 加载过程

  • DNS 解析：域名 -> IP地址
  • 浏览器根据 IP 地址向服务器发起 http 请求
  • 服务器处理 http 请求，并返回给浏览器

1.3 渲染过程

  • 根据 HTML 代码生成 DOM Tree
  • 根据 CSS 代码生成 CSSOM
  • 将 DOM Tree 和 CSSOM 整合形成 Render Tree
  • 根据 Render Tree 渲染页面
  • 遇到 < script > 则暂停渲染，优先加载并执行 JS 代码，完成再继续
  • 直至把 Render Tree 渲染完成

2. 网页加载和渲染的相关面试题

2.1 从输入 url 到渲染出页面的整个过程

  • 下载资源：各个资源类型，下载过程
  • 渲染页面：结合 HTML、CSS、JavaScript 图片等

2.2 window.onload 和 DOMContentLoaded 的区别

  • window.onload 资源全部加载完才会执行，包括图片
  • DOMContentLoaded DOM 渲染完成即可，图片可能尚未下载

3. 性能优化

  • 是一个综合性问题，没有标准答案，但要求尽量全面
  • 手写防抖、节流

3.1 性能优化原则

  • 多使用内存、缓存或其他方法
  • 减少 CPU 计算量，减少网络加载耗时
  • （适用于所有编程的性能优化——空间换时间）

3.2 从何入手

  • 让加载更快
    • 减少代码体积：压缩代码
    • 减少访问次数：合并代码，SSR 服务器端渲染，缓存
    • 使用更快的网络：CDN
  • 让渲染更快
    • CSS 放在 head，JS 放在 body 下面
    • 尽早开始执行 JS，用 DOMContentLoaded 触发
    • 懒加载（图片懒加载，上滑加载更多）
    • 对 DOM 查询进行缓存
    • 频繁 DOM 操作，合并到一起插入 DOM 结构
    • 节流 throttle，防抖 debounce

3.4 缓存

  • 静态资源加 hash 后缀，根据文件内容计算 hash
  • 文件内容不变，则 hash 不变，则 url 不变
  • url 和稳健不变，则会自动触发 http 缓存机制，返回 304

3.6 SSR

  • 服务器端渲染：将网页和数据一起加载，一起渲染
  • 非 SSR （前后端分离）：先加载网页，再加载数据，再渲染数据
  • 早先的 JSP ASP PHP，现在的 Vue React SSR

4. 防抖 debounce

  • 监听一个输入框，文字变化后触发 change 事件
  • 直接用 keyup 事件，则会频繁触发 change 事件
  • 防抖：用户输入结束或暂停时，才会触发 change 事件

5. 节流 throttle

  • 拖拽一个元素时，要随时拿到该元素被拖拽的位置
  • 直接用 drag 事件，则会频繁触发，很容易导致卡顿
  • 节流：无论拖拽速度多快，都会每隔 100ms 触发一次

6. 安全

问题：常见的 web 前端攻击方式有哪些？

  • XSS 跨站请求共计
  • XSRF 跨站请求伪造

6.1 XSS 跨站请求攻击

  • 一个博客网站，我发表一篇博客，其中嵌入 < script > 脚本
  • 脚本内容：获取 cookie，发送到我的服务器（服务器配合跨域）
  • 发布这篇博客，有人查看它，我轻松收割访问者的 cookie

6.2 XSS 跨站请求预防

  • 替换特殊字符，如 < 变为 & lt; > 变为 & gt;
  • < script >变为 & lt;script& gt;，直接显示，而不会作为脚本执行
  • 前端要替换，后端也要替换，都做总不会有错

6.3 XSRF 跨站请求伪造攻击

  • 你正在购物，看中了某个商品，商品 id 是 100
  • 付费接口是 xxx.com/pay?id=100，但没有任何验证
  • 我是攻击者，我看中了一个商品，id 是 200
  • 我向你发送一封电子邮件，邮件标题很吸引人
  • 但邮件正文隐藏着 < img src =xxx.com/pay?id=200 />
  • 你一查看邮件，就帮我购买了 id 是 200 的商品

6.4 XSRF 跨站请求伪造预防

  • 使用 post 接口
  • 增加验证，例如密码、短信验证码、指纹等
