1. HTML题

1.1 什么是 HTML 语义化

  1. 让人更容易读懂代码（增加代码可读性）
  2. 让搜索引擎更容易读懂（SEO）

1.2 块状元素 & 内联元素

  • display: block/table; 有：div h1 h2… table ul ol p 等
  • display:inline/inline-block; 有：span img input button 等

2. CSS题

2.1 布局

2.1.1 盒子模型宽度的计算

  • offsetWidth = （内容宽度 + 内边距 + 边框），无外边距

  • 答案：122px

  • 如何让 offsetWidth 等于 100px?

  • 答案：在 div1 属性中添加“ box-sizing: border-box;”

2.1.2 margin 纵向重叠问题

答案：15px

  • 相邻元素的 margin-top 和 margin-bottom 会发生重叠
  • 空白内容的 p 标签也会重叠
  • 大数值覆盖小数值

2.1.3 margin 负值问题

  • margin-top 和 margin-left 负值，元素向上、向左移动

  • margin-right 负值，右侧元素左移，自身不受影响

  • margin-bottom 负值，下方元素上移，自身不受影响

2.1.4 BFC 理解与应用

什么是 BFC ？如何应用？

  • Block format context，块级格式化上下文

  • 一块独立渲染区域，内部元素的渲染不会影响边界以外的元素

形成 BFC 的常见条件

  • float 不是 none
  • position 是 absolute 或 fixed
  • overflow 不是 visible
  • display 是 flex inline-block 等

BFC 的常见应用

  • 清除浮动

2.1.5 float 布局

  • 如何实现圣杯布局和双飞翼布局
  • 手写 clearfix

圣杯布局和双飞翼布局的目的

  • 三栏布局，中间一栏最先加载和渲染（内容最重要）
  • 两侧内容固定，中间内容随着宽度自适应
  • 一般用于 PC 网页

圣杯布局和双飞翼布局的技术总结

  • 使用 float 布局
  • 两侧使用 margin 负值，以便和中间内容横向重叠
  • 防止中间内容被两侧覆盖，一个用 padding 留白（圣杯布局），一个用 margin 留白（双飞翼布局）

手写 clearfix

2.1.6 flex 布局

flex 实现一个三点的色子

常用语法

  • flex-direction 主轴的方向（横向或者纵向）
  • justify-content 主轴的对齐方式（两端对齐，居中对齐等）
  • align-items 交叉轴对齐方式（和主轴垂直的轴。开始，结束，居中对齐等）
  • flex-wrap 换行
  • align-self 子元素在交叉轴的对齐方式（开始，结束，居中对齐等）

2.2 定位

2.2.1 absolute 和 relative 分别依据什么定位

  • relative 依据自身定位
  • absolute 依据最近一层的定位元素定位

定位元素

  • absolute relative fixed
  • body

2.2.2 居中对齐有哪些实现方式

水平居中

  • inline 元素：text-align: center
  • block 元素：margin: auto
  • absolute 元素：left: 50% + margin-left 负值

垂直居中

  • inline 元素：line-height 的值等于 height 值
  • absolute 元素：top: 50% + margin-top 负值
  • absolute 元素：transform(-50%, -50%)
  • absolute 元素：top, left, bottom, right = 0 + margin: auto
  • 此方法适用于保证兼容性，子元素尺寸是未知的

2.3 图文样式

line-height 如何继承

答案：p 标签的行高是40. (20px*200%)

  • 写具体数值，如 30px，则继承该值
  • 写比例，如 2 / 1.5，则继承该比例 （子元素的 font-size * 该比例）
  • 写百分比，如 200%，则继承计算出来的值 （父元素的 font-size * 该百分比）

2.4 响应式

2.4.1 rem 是什么

rem 是一个长度单位

  • px，绝对长度单位，最常用
  • em，相对长度单位，相对于父元素，不常用
  • rem，相对长度单位，相对于根元素，常用于响应式布局

2.4.2 响应式布局的常见方案

  • media-query，根据不同的屏幕宽度设置根元素 font-size
  • rem，基于根元素的相对单位

2.4.5 vw/vh

  • rem 的弊端：阶梯性
  • 网页视口尺寸
    • window.screen.height // 屏幕高度
    • window.innerHeight // 网页视口高度
    • doucument.body.clientHeight // body高度
  • vw/vh
    • vh 网页视口高度的 1/100
    • vm 网页视口宽度的 1/100
    • vmax 取两者最大值；vmin 取两者最小值
